    Tesla Gateway Security Research

    Complete reverse engineering of Tesla Gateway ECU configuration system

    What Is This?

    This repository documents comprehensive security research on the Tesla Gateway ECU ‚Äî the PowerPC-based controller that manages vehicle configuration, CAN bus routing, and feature enablement across Model S/X/3/Y vehicles.

    Key discoveries: - 662 Gateway configurations extracted and validated (CRC-8, polynomial 0x2F) - Two-tier security model discovered: UDP-accessible configs vs. Hermes-authenticated - Tesla Odin service tool database obtained (unhashed, 2,988 Python scripts) - Complete firmware analysis: 6MB PowerPC binary, 37,702 strings, 6,647 CAN messages - Multiple attack vectors documented with working proof-of-concepts

    Research scope: Gateway ECU (primary), Autopilot ECU, MCU2, authentication systems, update mechanisms

    üî¥ Critical Findings

    1. Gateway Two-Tier Security Model ‚úÖ Verified

    Finding: Gateway configs split into three access tiers:

    1. Insecure (UDP port 3500, no auth):
    2. Map region, autopilot trial timer, ECU map version
    3. Marked accessLevel: "UDP" in Odin database

    4. Anyone on vehicle network can modify

    5. Secure (Hermes-authenticated):

    6. VIN, country code, supercharger access
    7. Requires Tesla backend authentication via WSS:443

    8. accessId 7-43 in Odin database

    9. Hardware-locked (Gateway-only, fused):

    10. Debug security level (LC_FACTORY vs LC_GATED)
    11. Tied to MPC5748G hardware fuses
    12. Cannot change after chip fusing

    Evidence: Odin database (accessLevel flags), 662 configs extracted, CRC algorithm validated
    Impact: Insecure configs modifiable by anyone with network access
    Details: GATEWAY-SECURITY.md

    2. Odin Service Tool Config Database (Unhashed) ‚úÖ Verified

    Finding: Tesla's Odin service tool configuration database obtained before encryption was added, revealing complete config system architecture.

    Contents: - 2,988 Python scripts extracted from Model 3/Y firmware - Unhashed JSON database mapping accessId ‚Üí config names ‚Üí enum values - Config read API: get_vehicle_configuration(access_id=INTEGER) ‚Äî NO authentication for normal access levels - Complete list of which configs require elevated permissions

    Evidence: JSON file obtained, scripts extracted from firmware
    Impact: Complete map of vehicle configuration system
    Details: ODIN-CONFIG-DATABASE.md

    3. CAN Flood Opens Emergency Updater ‚ö†Ô∏è Inferred

    Finding: Flooding CAN bus with specific message IDs opens TCP port 25956 on Gateway within 10-30 seconds, providing unauthenticated firmware update interface.

    Method: - CAN ID 0x3C2 @ 10,000 msg/sec (data: 49 65 00 00 00 00 00 00) - CAN ID 0x622 @ 33 msg/sec (UDS tester-present) - Opens port 25956 with 4 commands: help, set_handshake, install, status

    Evidence: Working Python script, timing analysis, reported 98% success rate
    Hardware: PCAN USB adapter (~$50), OBD-II access
    Details: ATTACK-CAN-FLOOD.md

    4. Service Mode Requires Backend Validation ‚ö†Ô∏è Inferred

    Finding: Service mode authentication is NOT a simple local PIN check ‚Äî it uses DoIP (Diagnostic over IP) + Protobuf signed commands + backend validation.

    Evidence: - D-Bus method analysis: setServicePIN() at QtCar:0x655ec0 - No local PIN comparison found in binaries - Signed command infrastructure: optional_signed_cmd_service_mode - doip-gateway user has special D-Bus permissions

    Impact: Cannot bypass service mode offline without signed commands
    Offline behavior: Unknown (requires live testing)
    Details: AUTH-SERVICE-MODE.md

    5. Complete Firmware Disassembly ‚úÖ Verified

    Finding: Complete Gateway firmware reverse-engineered: - 6,029,152 byte PowerPC MPC5748G binary - 1.5M line disassembly generated - 37,702 strings extracted (ASCII + UTF-16) - 6,647 CAN message entries documented - 21,000+ config metadata entries located at 0x403000

    Evidence: Binary dumps, disassembly, string databases, CRC validation
    Details: GATEWAY-FIRMWARE.md
