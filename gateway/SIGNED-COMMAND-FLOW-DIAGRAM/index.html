    Gateway Config Write Security Flow - Visual Diagram

    Purpose: Visual representation of Gateway's config write validation
    Date: 2026-02-03
    Related: SIGNED-COMMAND-ANALYSIS.md

    Flow 1: Insecure Config Write (No Auth Required)

    ┌──────────────────────────────────────────────────────────────────────┐
│                    INSECURE CONFIG WRITE                             │
│              (e.g., mapRegion, superchargingAccess)                  │
└──────────────────────────────────────────────────────────────────────┘

  Attacker Laptop (192.168.90.100)
         │
         │ Python/socat UDP packet:
         │ 0c 00 42 01  (SET_CONFIG mapRegion = EU)
         │
         ▼
  ┌─────────────────┐
  │  UDP:3500       │
  │  Gateway ECU    │
  │  192.168.90.102 │
  └────────┬────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Parse Packet:                       │
  │  - Opcode: 0x0c (SET_CONFIG)         │
  │  - Flags:  0x00 (ignored)            │
  │  - ID:     0x0042 (mapRegion)        │
  │  - Value:  0x01 (EU)                 │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Lookup Config ID in Secure Table:   │
  │  is_secure_config(0x42)?             │
  └────────┬─────────────────────────────┘
           │
           ├─────> NO (0x42 is INSECURE)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Write to Flash:                     │
  │  - Offset: 0x19000 + index           │
  │  - Format: [CRC][Len][ID][Data]      │
  │  - New value: 0x01                   │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Return Success:                     │
  │  Echo packet: 0c 00 42 01            │
  └────────┬─────────────────────────────┘
           │
           ▼
  Attacker Laptop
         │
         └─> [✓] Config changed! No auth needed!


  Time elapsed: ~50ms
  Security checks: ❌ NONE (insecure config)
  Detection: ❌ NO (not logged, not monitored)

    Flow 2: Secure Config Write (No Auth) - REJECTED

    ┌──────────────────────────────────────────────────────────────────────┐
│                     SECURE CONFIG WRITE (NO AUTH)                    │
│                            (e.g., VIN)                               │
└──────────────────────────────────────────────────────────────────────┘

  Attacker Laptop (192.168.90.100)
         │
         │ UDP packet:
         │ 0c 00 00 5a 45 4e 4e 5f 54 45 53 54 5f 56 49 4e
         │ (SET_CONFIG VIN = "ZENN_TEST_VIN")
         │
         ▼
  ┌─────────────────┐
  │  UDP:3500       │
  │  Gateway ECU    │
  └────────┬────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Parse Packet:                       │
  │  - Opcode: 0x0c                      │
  │  - ID:     0x0000 (VIN)              │
  │  - Value:  "ZENN_TEST_VIN"           │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Lookup in Secure Table:             │
  │  is_secure_config(0x00)?             │
  └────────┬─────────────────────────────┘
           │
           ├─────> YES (VIN is SECURE)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Check Session Authentication:       │
  │  if (session_authenticated) {...}    │
  └────────┬─────────────────────────────┘
           │
           ├─────> FALSE (no Hermes session)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Reject Write:                       │
  │  Return 0xff (permission denied)     │
  └────────┬─────────────────────────────┘
           │
           ▼
  Attacker Laptop
         │
         └─> [✗] Rejected! Need Hermes auth


  Time elapsed: ~10ms (quick rejection)
  Security checks: ✅ Config classification, ✅ Session auth
  Detection: ⚠️ May be logged (suspicious VIN write attempt)

    Flow 3: Secure Config Write (WITH Hermes Auth) - SUCCESS

    ┌──────────────────────────────────────────────────────────────────────┐
│               SECURE CONFIG WRITE (HERMES AUTHENTICATED)             │
│                    (Tesla Technician Workflow)                       │
└──────────────────────────────────────────────────────────────────────┘

  Tesla Technician
         │
         │ 1. hermes_client --enable-phone-home
         │
         ▼
  ┌───────────────────────────────────────┐
  │  Hermes Backend                       │
  │  hermes-api.*.vn.cloud.tesla.com:443  │
  └────────┬──────────────────────────────┘
           │
           │ 2. WebSocket Secure (WSS) connection
           │    + mTLS (client cert: /var/lib/car_creds/car.crt)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Authenticate Technician:            │
  │  - Username: john.doe@tesla.com      │
  │  - Password: ********                │
  │  - 2FA code: 123456                  │
  └────────┬─────────────────────────────┘
           │
           ├─────> ✓ Valid credentials
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Send AUTH_GRANTED (Protobuf):       │
  │  {                                   │
  │    vin: "5YJSA1E26HF888888",         │
  │    timestamp: 1738583123,            │
  │    permissions: ["WRITE_VIN", ...],  │
  │    timeout: 1800  // 30 min          │
  │  }                                   │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌─────────────────┐
  │  Gateway ECU    │
  └────────┬────────┘
           │
           │ 3. Receive AUTH_GRANTED
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Set Session State:                  │
  │  session_authenticated = true        │
  │  auth_timeout = now() + 1800         │
  │  cached_permissions = [...]          │
  └────────┬─────────────────────────────┘
           │
           │ 4. Technician runs gw-diag tool:
           │    gw-diag write 0x0000 --value "5YJSA1E26HF999999"
           │
           ▼
  ┌──────────────────────────────────────┐
  │  gw-diag Tool                        │
  │  (on technician laptop or MCU)       │
  └────────┬─────────────────────────────┘
           │
           │ 5. Send UDP packet (NO SIGNATURE!):
           │    0c 00 00 35 59 4a 53 41 31 45 32 36 48 46 39 39 39 39 39 39
           │
           ▼
  ┌─────────────────┐
  │  UDP:3500       │
  │  Gateway ECU    │
  └────────┬────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Parse Packet:                       │
  │  - Opcode: 0x0c                      │
  │  - ID:     0x0000 (VIN)              │
  │  - Value:  "5YJSA1E26HF999999"       │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Lookup in Secure Table:             │
  │  is_secure_config(0x00)?             │
  └────────┬─────────────────────────────┘
           │
           ├─────> YES (VIN is SECURE)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Check Session:                      │
  │  if (session_authenticated &&        │
  │      time() < auth_timeout) {...}    │
  └────────┬─────────────────────────────┘
           │
           ├─────> TRUE (authenticated session active)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Check Permissions:                  │
  │  if ("WRITE_VIN" in permissions) {   │
  │    allow();                          │
  │  }                                   │
  └────────┬─────────────────────────────┘
           │
           ├─────> TRUE (permission granted)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Write to Flash:                     │
  │  - Calculate CRC-8                   │
  │  - Format: [CRC][17][0x00][VIN]      │
  │  - Offset: 0x19xxx                   │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Log Change (Audit):                 │
  │  - Technician: john.doe@tesla.com    │
  │  - Config: VIN (0x0000)              │
  │  - Old: 5YJSA1E26HF888888            │
  │  - New: 5YJSA1E26HF999999            │
  │  - Timestamp: 1738583123             │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Return Success (Echo):              │
  │  0c 00 00 35 59 4a 53 41 31 45 ...   │
  └────────┬─────────────────────────────┘
           │
           ▼
  Technician
         │
         └─> [✓] VIN changed successfully!
             Logged to backend (full audit trail)


  Time elapsed: ~5-10 minutes (Hermes auth + command)
  Security checks: ✅ mTLS, ✅ Credentials, ✅ Session, ✅ Permissions
  Detection: ✅ YES (full audit log uploaded to backend)

    Flow 4: UnlockSwitch Attempt (FAILS)

    ┌──────────────────────────────────────────────────────────────────────┐
│              UNLOCKSWITCH + VIN WRITE (ATTEMPT)                      │
│               (Hypothesis: Factory mode bypasses auth)               │
└──────────────────────────────────────────────────────────────────────┘

  Attacker Laptop
         │
         │ Step 1: Send UnlockSwitch
         │ 18 ba bb a0 ad
         │
         ▼
  ┌─────────────────┐
  │  Gateway ECU    │
  └────────┬────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Handle UnlockSwitch:                │
  │  if (magic == 0xBABBA0AD) {          │
  │    factory_mode_active = true;       │
  │    emergency_port_enabled = true;    │
  │    return 0x1801;  // ACK            │
  │  }                                   │
  └────────┬─────────────────────────────┘
           │
           ├─────> Factory mode ACTIVATED
           │       (emergency_port UDP:25956 now listening)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Return Acknowledgement:             │
  │  18 01 (factory mode enabled)        │
  └────────┬─────────────────────────────┘
           │
           ▼
  Attacker Laptop
         │
         │ Step 2: Try VIN write
         │ 0c 00 00 5a 45 4e 4e 5f 54 45 53 54 5f 56 49 4e
         │
         ▼
  ┌─────────────────┐
  │  Gateway ECU    │
  └────────┬────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Parse SET_CONFIG VIN                │
  │  Config ID: 0x0000 (secure)          │
  └────────┬─────────────────────────────┘
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Check Security:                     │
  │  if (is_secure_config(0x00)) {       │
  │    // Factory mode does NOT affect   │
  │    // secure config protection!      │
  │    if (!session_authenticated) {     │
  │      return 0xff;  // REJECT         │
  │    }                                 │
  │  }                                   │
  └────────┬─────────────────────────────┘
           │
           ├─────> session_authenticated = false
           │       (UnlockSwitch did NOT set this!)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Return Rejection:                   │
  │  0xff (still need Hermes auth!)      │
  └────────┬─────────────────────────────┘
           │
           ▼
  Attacker Laptop
         │
         └─> [✗] FAILED! UnlockSwitch doesn't help!


  Conclusion: factory_mode_active ≠ session_authenticated
              UnlockSwitch enables diagnostics, NOT secure writes

    Flow 5: JTAG Bypass (Physical Attack)

    ┌──────────────────────────────────────────────────────────────────────┐
│                    JTAG FLASH MODIFICATION                           │
│                 (Physical Security Bypass)                           │
└──────────────────────────────────────────────────────────────────────┘

  Attacker with Physical Access
         │
         │ 1. Remove Gateway ECU from vehicle
         │
         ▼
  ┌──────────────────────────────────────┐
  │  Gateway MPC5748G SPC Chip           │
  │  (BGA package, no exposed pins)      │
  └────────┬─────────────────────────────┘
           │
           │ 2. BGA rework (solder JTAG pins)
           │    Cost: $200-500 (BGA rework service)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  JTAG Debugger Connected             │
  │  (e.g., Segger J-Link, $600-4700)    │
  └────────┬─────────────────────────────┘
           │
           │ 3. Read flash dump (6 MB)
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Flash Memory (6 MB PowerPC fw)      │
  │  - Offset 0x19000: Config region     │
  │  - Format: [CRC][Len][ID][Data]      │
  └────────┬─────────────────────────────┘
           │
           ▼
  Attacker's Computer
         │
         │ 4. Locate VIN entry
         │    Search for: [??][11][00 00][17-byte VIN]
         │
         │ 5. Modify VIN
         │    Old: 5YJSA1E26HF888888
         │    New: 5YJSA1E26HF999999
         │
         │ 6. Calculate new CRC-8 (poly 0x2F)
         │    CRC = crc8([11][00 00][new VIN])
         │
         ▼
  ┌──────────────────────────────────────┐
  │  Modified Flash Image:               │
  │  [NEW_CRC][11][00 00][5YJSA1E26HF...]│
  └────────┬─────────────────────────────┘
           │
           │ 7. Write back to flash via JTAG
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Flash Memory (modified)             │
  │  VIN changed at offset 0x19xxx       │
  └────────┬─────────────────────────────┘
           │
           │ 8. Reboot Gateway
           │
           ▼
  ┌──────────────────────────────────────┐
  │  Gateway Boots:                      │
  │  - Loads config from flash           │
  │  - New VIN: 5YJSA1E26HF999999        │
  │  - NO signature check on configs!    │
  │  - ALL software security bypassed    │
  └────────┬─────────────────────────────┘
           │
           ▼
  Vehicle
         │
         └─> [✓] VIN changed! Identity cloned!

             ⚠️  Detection risks:
             - Firmware hash check (configs 0x25/0x26)
             - Backend VIN mismatch alerts
             - Hermes cert binding (VIN in cert CN)


  Time: 2-4 hours (disassembly + BGA + programming)
  Cost: $600-5200 (JTAG debugger + BGA rework)
  Skill: HIGH (electronics, PowerPC, reverse engineering)
  Success Rate: ✅ 100% (if done correctly)
  Detection: 🟡 MEDIUM (firmware hash may alert backend)

    Key Differences Summary

    Method Packets Auth Success Time Cost Risk
    Insecure Config UDP:3500 ❌ None ✅ 100% 50ms $0 ✅ LOW (no logs)
    Secure Config (no auth) UDP:3500 ❌ None ❌ 0% 10ms $0 ⚠️ LOW (rejected)
    Secure Config (Hermes) UDP:3500 + WSS:443 ✅ mTLS + creds ✅ 100% 5-10min $0 🔴 HIGH (full audit)
    UnlockSwitch UDP:3500 ❌ None ❌ 0% 1s $0 ⚠️ LOW (doesn't work)
    JTAG Flash Hardware ❌ None ✅ 100% 2-4h $600-5200 🟡 MEDIUM (hash check)
